Interface overview

The One-Page-Summary

On this page, you can take a short overview over the properties and methods a plugin must have and what they are supposed to do.

Interface Version: 1.0.0.0

Properties:

string Name;
Read-only; Contains the name of the plugin, which is shown in the Control Panel.

bool Enabled;
Read-only; Indicates, if a plugin is available after calling the Enable() method.

Methods:

void Initialize();
Called, when searching for and loading plugins. Any initialization work should be done here.

bool Enable();
Called, when all plugins are loaded. Returns a boolean value indicating if the plugin is available.

PluginSettingsCollection GetSettings();
Called, when the plugin is selected in the Control panel. Returns a PluginSettingsCollection with the plugin's settings.

void SetSettings(PluginSettingsCollection settings);
Called, when the plugin's settings are clear-cut and the Control Panel is ready to save the work.

object SaveRecordingScript();
Called, when the user clicks the "OK" button. The plugin should process its settings and do any save work, mostly writing a recording script.

void Dispose();
Called, when the Control Panel is being closed. Any cleaning up work should be done here.

Events:

Currently none implemented.
