The OTRAPI

Sticking LEGO stones together

The OTR Remote API is housed in the OTRAPI.dll and contains all needed code for plugin and main application to communicate with each other. It also contains a "skeleton" for the plugin, which is needed for plugins to be compatible with OTR Remote. Without it, the application would never know, which method to call in which situation. You can see it as general rules for the plugin developer.

The "skeleton" - called interface - does, however, not influence the developer in any of his programming conventions, it only makes sure, that the methods needed for communicating with OTR Remote are consistent.
